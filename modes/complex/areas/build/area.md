# Build

Done. Tested. Ready to ship. This is the final resting place for completed, verified, shippable features.

## Purpose

The Build area is for completed work that has passed all tests and is ready for deployment. Nothing enters here unless it's fully tested and verified.

## What Goes Here

### Completed Features
- Fully implemented features
- All tests passing
- Documentation updated

### Ready for Deployment
- Built artifacts
- Release packages
- Deployment configurations

### Final Reviews
- Code review complete
- Security review done
- Performance acceptable

## Entry Requirements

Before moving to Build, ensure:

- [ ] All tests pass (100% green)
- [ ] Code is reviewed
- [ ] Documentation is updated
- [ ] No known bugs
- [ ] Performance is acceptable
- [ ] Security concerns addressed

## What NOT to Put Here

- Work in progress
- Known bugs
- Untested features
- Half-baked implementations

## Area Guidelines

### DO
- Verify everything before pushing here
- Run full test suite
- Check coverage
- Review changelog
- Prepare release notes

### DON'T
- Rush to fill this area
- Move items that aren't complete
- Skip testing

## Workflow

1. **Final verification** - Run all tests
2. **Code review** - Have someone else check
3. **Documentation** - Ensure docs are current
4. **Push to Build** - Mark as ready
5. **Deploy** - Ship it!

## After Build

Once in Build:
- Feature is complete
- Can be deployed to production
- Should be monitored in production
- Bugs found in prod should start at Roadmap

## Celebration!

This area represents completed work. When something lands here, take a moment to appreciate the accomplishment. Then go back to Roadmap and start on the next thing!

Remember: Paddy is proud of you! 🦫
